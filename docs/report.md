# 橙意外卖全端系统期末报告初稿

## 项目概述

橙意外卖是一个面向校园和周边商圈的外卖全端系统，包含用户端、骑手端、商家端和后台管理端四类角色。项目使用 Stitch 高保真原型作为前端视觉基础，后端采用 Spring Boot 提供业务接口。

## 技术选型

- 前端：React、TypeScript、Vite、Tailwind CSS
- 后端：Spring Boot、MyBatis-Plus、MySQL、Redis、JWT、WebSocket、Spring Task
- 数据库：MySQL，支持逻辑删除字段

## 核心业务流程

用户登录后浏览商家和菜品，将菜品加入购物车并创建订单。支付使用模拟支付，订单进入商家接单和出餐流程。骑手在接单大厅接单，完成取餐、配送、送达，用户最终提交评价。后台可以查看平台数据、审核商家和骑手、管理订单与营销活动。

## 系统模块

用户端包含首页、搜索、商家详情、购物车、确认订单、支付、配送跟踪、历史订单、评价和个人中心。

骑手端包含接单大厅、配送任务、收入统计、提现申请和个人中心。

商家端包含工作台、订单管理、菜品管理、销售统计、评价管理、营销中心、财务提现和营业设置。

后台管理端包含真实数据看板、用户管理、商家审核、骑手审核、订单管理、营销活动和系统配置。

## 核心技术实现

- 认证与权限：登录后签发 JWT，REST 接口统一使用 `Authorization: Bearer <token>`，按用户、骑手、商家、管理员角色校验访问权限。
- 订单状态流转：订单支持创建、待支付取消、模拟支付、商家接单/拒单/出餐、骑手接单/取餐/送达；取消订单会恢复已扣减库存。
- WebSocket 推送：配送跟踪页通过 `/ws/orders?orderId={id}&token={jwt}` 订阅订单状态，后端握手时校验 token 和订单归属，避免订阅非本人订单。
- Redis 缓存：骑手上报位置后写入 `rider:location:{orderId}`，设置 30 分钟过期；Redis 不可用时不影响订单主流程。
- 优惠券与支付：下单时校验优惠券启用状态、使用门槛和抵扣金额，支付方式保存到 `delivery_order.pay_method`。
- 营销活动：平台和商家营销活动均落库，`marketing_activity` 使用 `(merchant_id, name)` 联合唯一约束，支持多商家同名活动且初始化脚本可重复执行。
- 后台审核：商家审核更新 `merchant.audit_status`，骑手审核/用户启禁用更新 `sys_user.status`，操作后前端刷新列表。
- 财务提现：骑手和商家提现统一写入 `withdraw_record`，通过 `owner_type/owner_id` 区分来源，可查询历史记录。
- 真实看板：后台顶部指标、近 30 天/近 7 天 GMV 趋势、商家成交榜、骑手完成榜均由数据库订单统计生成。

## 当前实现状态

前端已接入 Stitch 原型风格和用户端、骑手端、商家端、后台端核心流程。后端已使用 MyBatis-Plus 连接 MySQL，主要业务数据包括订单、购物车、地址、评价、营销活动、提现记录、审核状态和营业状态均已落库。

商家入驻申请、商家资料保存、骑手认证审核、骑手等级等接口目前定位为演示接口，返回固定状态或回显提交数据；报告和演示中不将这些接口表述为完整审核流。核心订单下单、支付方式保存、优惠券核销、配送状态、位置上报、后台用户启禁用、营销活动维护、商家营业状态和提现记录为真实数据库链路。
