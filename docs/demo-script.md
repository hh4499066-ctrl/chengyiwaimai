# 5-8 分钟演示脚本

## 0:00-0:40 项目入口

打开前端 `http://localhost:3000`，说明橙意外卖包含用户端、骑手端、商家端和后台管理端，前端延续 Stitch 视觉风格，后端提供真实数据库接口。

## 0:40-2:20 用户下单

使用用户账号 `13800000001` 登录。进入首页后展示商家列表来自 `GET /merchants`，点击商家进入详情页，说明菜品来自 `GET /merchants/{merchantId}/dishes`。

点击菜品加号，触发 `POST /customer/cart`，进入购物车后说明购物车优先读取 `GET /customer/cart`，接口失败时前端保留 Mock fallback 以保证演示不中断。

进入确认订单页，点击提交订单，触发 `POST /orders`。强调后端返回的是 `Order` 对象，订单 ID 字段是 `id`。

## 2:20-3:10 支付与状态

进入支付页点击确认支付，触发 `POST /orders/{orderId}/pay`，订单状态从 `待支付` 变为 `待商家接单`。支付成功页展示用户侧结果。

## 3:10-4:30 商家端

切换到商家端，展示工作台、订单管理、菜品管理和评价模块。说明后端订单状态规则是：`待商家接单 -> 商家已接单 / 已取消`，然后 `商家已接单 -> 商家已出餐`。

## 4:30-5:40 骑手端

切换到骑手端，展示接单大厅、配送任务和收入统计。说明骑手侧状态为：`商家已出餐 -> 骑手已接单 -> 骑手已取餐 -> 已完成`。

## 5:40-6:40 用户评价与历史订单

回到用户端，查看配送跟踪、提交评价，并进入历史订单页。说明历史订单优先调用 `GET /orders`。

## 6:40-7:40 后台管理与异常控制

切换后台管理端，展示数据看板、商家审核、骑手审核和订单监控。补充说明：登录角色来自 `sys_user` 表，订单不存在会返回错误，待支付订单超过 15 分钟会被定时任务自动取消。

## 7:40-8:00 收尾

总结技术点：React + Vite 前端、Spring Boot 后端、MyBatis-Plus 数据库访问、统一异常处理、订单状态校验、WebSocket 订单状态推送和 Spring Task 超时取消。
