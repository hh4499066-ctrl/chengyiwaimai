# 5-8 分钟演示脚本

## 0:00-0:40 项目入口

打开 `http://localhost:3000`，说明橙意外卖包含用户端、骑手端、商家端和后台管理端，前端保留 Stitch 橙色卡片化视觉，后端提供真实 REST、JWT 和 WebSocket。

## 0:40-2:20 用户下单

用用户账号登录。演示首页定位切换、消息入口、分类筛选、综合排序、销量优先、距离最近和搜索页筛选。进入商家详情后切换菜品分类，点击菜品卡片打开详情弹层并加入购物车。

## 2:20-3:20 购物车、确认订单和支付

进入购物车调整数量。确认订单页选择地址、优惠券并填写备注。提交后在支付页切换微信、支付宝、校园一卡通，点击确认支付，支付失败不会跳成功页。

## 3:20-4:30 商家端

商家登录后查看工作台真实统计和订单列表。演示接单、拒单、出餐。进入商品管理，展示真实菜品列表、新建、编辑、上下架和补库存。

## 4:30-5:40 骑手端

骑手登录后切换上线/离线。上线后抢单，进入当前任务，点击取餐、位置上报和送达。收入页展示 `/rider/income` 返回的今日收入、今日订单、累计收入，并提交提现申请。

## 5:40-6:40 配送跟踪和评价

用户回到配送跟踪页，查看订单状态、骑手信息、联系骑手/商家和 Redis 缓存的最新位置提示。订单完成后选择星级，输入评价并提交。

## 6:40-7:40 后台管理

管理员登录，查看 `/admin/dashboard` 真实数据。切换今日/累计并导出 CSV。进入用户、订单、营销、商家审核和骑手审核模块，展示真实列表或演示表单交互。

## 7:40-8:00 收尾

总结：REST 不再支持 query token，WebSocket 仍使用 `orderId + token` 握手并校验订单归属；统计按今日范围过滤；Redis 用于骑手位置缓存；前后端构建命令均通过。
